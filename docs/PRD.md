# KICK QUIZ — PRD v4.0 (피드백 반영)

## 1. Executive Summary
- **프로젝트명**: KICK QUIZ (킥퀴즈)
- **비전**: "매일매일 축구 덕후력을 갱신한다"
- **플랫폼**: 토스 인앱 최적화 미니앱 (웹뷰 규격, 375x812)
- **출시 목표**: 2026년 여름방학 시즌 내 단독 런칭
- **기획 기조**: 비용 0원 유지, 철저한 법적/플랫폼 규제 회피, 유저 고정 리텐션(재방문) 확보

## 2. 주요 결정 사항

### 피드백 반영 (v3→v4)
| 변경 | 이전 | 이후 |
|------|------|------|
| 퀴즈 문항 수 | 8문항 | **4문항** (초급2+고급2) |
| 승부차기 확률 | 50% | **33% 3방향** (좌/중/우) |
| 포인트 경제 | 정답 100P/오답 50P | 초급정답 50P/고급정답 100P/오답 20P |
| 선물상자 | 500P→P환급 | **300P→유틸리티 아이템**+Toss쿠폰 |
| 응원탭 | 단순 매치 표시 | **팬팀 선택**+승부예측(예정) |
| 매치 데이터 | 하드코딩 | **localStorage CRUD** (admin.html) |
| 질문 DB | K리그 포함 | **K리그 제외** 50문항 |
| 광고 | 없음 | **초급/고급 사이 광고슬롯** |
| 초대 시스템 | 미정 | **URL 파라미터 Option B** (MVP 가능) |

## 5. 친구 초대 플로우

```
[유저 A] 초대코드 생성 (클라이언트 해시)
    │
    ├──→ [오픈카톡 공유: "킥퀴즈 함께해요! 코드: KQ-ABCD"]
    │
    ▼
[유저 B] 앱 설치 → 초대코드 입력
    │
    ├──→ A: +300P (오픈카톡 증빙 → 수동 정산)
    └──→ B: +300P (자동 지급)
```
