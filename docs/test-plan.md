# 테스트 계획서

## 1. 테스트 범위
| 영역 | 테스트 유형 | 도구 |
|------|------------|------|
| 프론트엔드 UI | 단위 테스트 (컴포넌트) | Vitest |
| 백엔드 API | 통합 테스트 (엔드포인트) | fetch + tsx |
| E2E | 주요 사용자 플로우 | 수동 |

## 2. API 통합 테스트
- `backend/src/test/api-test.ts` 참고
- 총 9개 GET 엔드포인트 + 1개 POST 엔드포인트
- 실행: `cd backend && npx tsx src/test/api-test.ts`
